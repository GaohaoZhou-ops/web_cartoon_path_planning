# ROUTE/LAB 路径规划观测台

一个纯前端的交互式网格路径规划实验台。可设置一个起点、任意多个有序途径点和一个终点，并从五类共 24 种算法中选择编队，以统一逻辑时钟同步运行。

## 运行

```bash
npm install
npm run dev
```

也可以使用服务脚本按端口在后台启动、查询和停止：

```bash
./service.sh
./service.sh start 5173
./service.sh status
./service.sh status 5173
./service.sh stop 12345
./service.sh stop
```

直接运行 `./service.sh` 会自动启动或复用一个服务，然后列出当前工程所有运行中服务的 PID、端口、启动时间、已运行时间、访问地址和进程命令；默认从端口 `5173` 开始选择。使用 `stop PID` 可以只停止指定进程，不带进程号的 `stop` 则停止当前工程的全部服务实例。运行日志和 PID 会按端口保存在 `.service/` 目录中。

生产构建与测试：

```bash
npm run build
npm test
```

## 使用方式

- 可在左侧设置地图列数与行数，范围为 `5–100`；输入后会自动更新，起终点与途径点会迁移到有效网格，障碍将重新随机生成并保留一条完整可达路线。
- 用左侧工具设置起点、途径点、终点；障碍画笔支持按住拖动，右键可临时擦除。
- 途径点按照列表顺序依次经过，可在列表中上移、下移或删除。
- 可切换四邻域/八邻域，并控制是否禁止斜向穿越墙角。
- 点击“选择算法并开始”后，可按应用领域浏览、全选或清除某一类算法，也可全选、仅选基础算法或自由编成本轮算法队列；选中的算法读取同一份不可变地图快照。
- 播放栏支持暂停、统一单步、重新运行及 `0.5×–8×` 速度调整。
- 算法结束时，其卡片会按完成名次依次前移，并自动滚动到第一张未完成卡片；已经锁定的名次不会被后完成的算法打乱。同一逻辑 tick 内结束时，以算法编队顺序稳定排位。
- 全部算法结束后会自动滚动到终局统计区，以动画排名和四组独立横向图表对比扩展节点、计算耗时、完整路径代价与峰值队列；每张图按该项表现从最好到最差自上而下排列。

## 算法分类

- **静态网格搜索**：A*、Bidirectional A*、Theta*、Dijkstra、Breadth-First、Greedy Best-First。
- **动态环境与重新规划**：D* Lite、Field D*、LPA*、AD*。
- **游戏寻路**：JPS、JPS+、Flow Field、HPA*。
- **连续空间与机器人导航**：Hybrid A*、State Lattice、Fast Marching Method、RRT*、PRM。
- **局部避障与轨迹规划**：TEB、DWA、VFH、Artificial Potential Field、TrajOpt。

## 对比口径

- **扩展 / 工作量**：网格与图搜索统计有效关闭的状态；HPA* 还包含抽象层扩展，姿态规划器统计位姿状态，局部规划器统计轨迹、方向或目标函数候选评估。不同状态空间的工作量适合观察行为，不代表完全相同的计算单位。
- **计算耗时**：仅累计在线搜索与路径回溯的 CPU 时间，不含 JPS+ 静态表预处理、动画等待、暂停和 Canvas 绘制。
- **路径代价**：直移为 `1`，斜移为 `√2`；BFS 优先最少移动步数，因此八邻域下不保证最低几何代价。
- **跳点算法**：JPS 与 JPS+ 仅把实际跳点计为扩展节点；最终路径会展开为连续网格步，因此步数与其他网格算法口径一致。
- **Flow Field**：会生成目标可达区域的完整积分场，扩展量与只规划一条路线的算法不是同一工作量口径。
- **Theta\***：保留任意角折线路径，以折线段的欧氏距离累计路径代价。
- **Field D\***：在 cell-center 对偶网格上执行真实边插值备份；四邻域下会退化为离散 D* 路径。
- **LPA\* / AD\***：使用 `g/rhs` 一致性状态；AD* 会按 `ε=2.5→2.0→1.5→1.0` 发布并修正有界解。核心还提供局部障碍变化的增量修复入口。
- **RRT\* / PRM**：使用确定性混合采样与固定预算，输出连续空间折线；预算失败仅表示本次树或路线图未连通，不是不可达证明。连续采样器需要启用斜向移动。
- **HPA\***：先在分块抽象图中确定层级走廊，再回到局部网格逐段细化；层级路径用于游戏地图上的快速近优查询。
- **Hybrid A\* / State Lattice**：在包含朝向的状态空间中搜索安全运动原语；当前页面采用点机器人教学模型，起始朝向由当前航段方向推断。
- **Fast Marching Method**：从航段终点反向求解到达时间场，再沿严格下降方向提取安全路径。
- **TEB / DWA / VFH / Artificial Potential Field / TrajOpt**：在静态地图快照上使用全局可行参考线，分别演示弹性带、动态窗口、方向直方图、势场和轨迹优化；这是二维点机器人的教学型适配，不包含移动障碍预测或真实车辆动力学。
- **连续与局部规划器**：Hybrid A*、State Lattice、RRT*、PRM、TEB、DWA、VFH、Artificial Potential Field 与 TrajOpt 需要启用斜向移动。
- 每个逻辑 tick 为每个尚未结束的算法分配一次有界工作单元；网格搜索通常展开一个状态，局部控制器会评估一批有界候选。提前完成的算法会冻结在最终状态。

实现采用 React + TypeScript + Canvas，路径搜索核心位于 `src/pathfinding.ts`，统一的可视化绘制位于 `src/GridCanvas.tsx`。
