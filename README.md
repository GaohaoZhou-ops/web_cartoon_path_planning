# ROUTE/LAB 路径规划观测台

一个纯前端的交互式网格路径规划实验台。可设置一个起点、任意多个有序途径点和一个终点，并以统一逻辑时钟同步运行 A*、JPS、Dijkstra、BFS 与 Greedy Best-First。

## 运行

```bash
npm install
npm run dev
```

也可以使用服务脚本按端口在后台启动、查询和停止：

```bash
./service.sh start 5173
./service.sh status 5173
./service.sh stop
```

运行日志和 PID 会按端口保存在 `.service/` 目录中；`stop` 无需端口号，会停止当前工程启动的全部服务实例。

生产构建与测试：

```bash
npm run build
npm test
```

## 使用方式

- 用左侧工具设置起点、途径点、终点；障碍画笔支持按住拖动，右键可临时擦除。
- 途径点按照列表顺序依次经过，可在列表中上移、下移或删除。
- 可切换四邻域/八邻域，并控制是否禁止斜向穿越墙角。
- 点击“锁定并开始”后，五个算法读取同一份不可变地图快照。
- 播放栏支持暂停、统一单步、重新运行及 `0.5×–8×` 速度调整。
- 算法结束时，其卡片会按完成名次依次前移，并自动滚动到第一张未完成卡片；已经锁定的名次不会被后完成的算法打乱。同一逻辑 tick 内结束时，以算法编队顺序稳定排位。
- 全部算法结束后会自动滚动到终局统计区，以动画排名和四组独立横向图表对比扩展节点、计算耗时、完整路径代价与峰值队列。

## 对比口径

- **扩展节点**：从 frontier 中有效取出并关闭的节点数，目标节点也计入。
- **计算耗时**：仅累计增量搜索与路径回溯的 CPU 时间，不含动画等待、暂停和 Canvas 绘制。
- **路径代价**：直移为 `1`，斜移为 `√2`；BFS 优先最少移动步数，因此八邻域下不保证最低几何代价。
- **JPS 跳点**：仅把实际跳点计为扩展节点；最终路径会展开为连续网格步，因此步数与其他算法口径一致。
- 每个逻辑 tick 为每个尚未结束的算法分配一次有效节点扩展；提前完成的算法会冻结在最终状态。

实现采用 React + TypeScript + Canvas，路径搜索核心位于 `src/pathfinding.ts`，统一的可视化绘制位于 `src/GridCanvas.tsx`。
